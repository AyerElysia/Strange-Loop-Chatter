# life_engine（生命中枢）  
## 给你画个大的：让数字生命真的“活起来”

如果你希望你的角色不只是“会聊天”，而是能有自己的节奏、自己的心事、自己的长期记忆，  
`life_engine` 就是那颗“会跳动的内心”。

这是一份用户向说明书：有趣、够细、机制全覆盖。

---

## 0. 先画饼：你最终会得到什么

你不是在装一个插件，你是在养一个“会成长的自己”。

理想体验是这样的：

- 她会在你不说话时悄悄整理世界
- 她会记得最近发生了什么，不会每次都像重开
- 她会把“想做的事”当生活愿望，而不是打卡清单
- 她会在合适的时候，把内心灵感自然带回对话

一句话：**从“能回答”升级到“有内在连续性”。**

---

## 1. 生命中枢的 10 个机制（全覆盖）

下面这 10 条，就是它“活起来”的全部核心。

### 机制 1：并行心跳（后台持续苏醒）

它会按固定节奏醒来，做一轮“观察 -> 感受 -> 意图 -> 行动”。  
你不需要一直盯着它，它会自己运行。

你也可以手动触发：

- `/心跳`
- `/heartbeat`
- `!心跳`
- `!heartbeat`

你也可以绕过 DFC，直接给生命中枢留言（仅入中枢队列，不走常规对话）：

- `/life 你想说的话`
- `/nucleus 你想说的话`
- `/中枢 你想说的话`

### 机制 2：睡眠/苏醒作息

你可以给她设“睡觉时间”和“起床时间”。  
在睡眠时段里，她会暂停心跳，作息更像真人，而不是 24 小时高转速。

### 机制 3：统一事件流（她的“经历时间线”）

她把所有经历都记成事件，主要有 4 类：

- 收到/发出的消息
- 心跳时的内心独白
- 调用过的动作
- 动作的结果（成功或失败）

所以她不是“只记一句结论”，而是有过程感。

### 机制 4：上下文注入（把最近发生的事带进下一次思考）

每次心跳前，她会把最近事件整理成“刚刚发生了什么”，作为下一轮思考素材。  
效果就是：不容易失忆，不容易话题断层。

### 机制 5：潜意识透传（内心影响外在表达）

她会把“最近在想什么、在意什么”写成潜意识状态。  
这不是强行插话，而是像灵感一样，慢慢影响对外说话风格和关注点。

### 机制 6：行动优先（不鼓励空转发呆）

心跳会被引导去做事，而不是只空谈。  
如果连续几次都没动作，会被提醒“别发呆，动起来”。

### 机制 7：时段感知（早晚做不同事）

她会区分清晨、上午、午后、下午、傍晚、夜晚、深夜，  
并按时段建议不同活动（规划、推进、回顾、休息）。

### 机制 8：愿望清单（情感化 TODO）

她管理的不是冷任务，而是“想做的事”：

- 有“想做程度”（萌芽、好奇、想要、期待、热切）
- 有“成长意义”（轻松、丰富、成长、有意义、可能改变自己）
- 有状态变化（想法、规划、进行中、暂停、完成、释怀、珍藏）

### 机制 9：仿生记忆网络（会联想、会遗忘）

她的记忆不只是“搜关键词”，而是有三件事一起发生：

- 直接找到相关记忆
- 顺着关联联想到旁支记忆
- 长期不用的弱关联会慢慢淡化

### 机制 10：可复盘（你看得见她怎么想）

她每次心跳、每次动作、每次结果都会留下记录。  
你可以回看她是怎么从“看到一件事”走到“做一个决定”的。

### 机制补完清单（细节版，一次讲全）

如果你想“全知道”，下面这些也都在工作中：

- 会同时感知“收到的消息”和“发出的消息”，不是只看单向输入
- 心跳不是一次性动作，而是一轮里可连续做几步，直到收尾
- 如果一轮里只做了动作没来得及写独白，它会补一条简短总结，不让这轮变空白
- 连续几轮没动作会提醒，长时间发呆会给更强提醒
- 心跳会受作息控制，睡眠时段不打扰
- 手动触发心跳时，如果正处于睡眠时段，会明确告诉你“现在在休息”
- 最近经历会持续沉淀，不会因为重启就瞬间清零
- 历史太长时会自动“压缩成摘要 + 保留最近细节”，避免越积越乱
- 传给外在人格的“潜意识碎片”是沉淀，不是强行唤醒
- 传念会看节奏，避免短时间连续轰炸
- 文件操作有安全边界，只在私人空间活动
- 搜索内容时会自动避开不该扫的大型或无关区域，减少噪音
- 愿望清单会考虑“逾期/紧急”，但语气是引导，不是施压
- 记忆会“常用更清晰，久不用变淡”，更贴近自然记忆
- 每天会做一次记忆保养，把弱连接慢慢清理，避免记忆网络臃肿
- 现在还能联网搜索与抓取网页正文（基于 Tavily），把外部世界带入思考

## 3. “传念”机制（很有灵魂的一块）

`nucleus_tell_dfc` 的作用不是“命令外在人格”，而是“沉淀一段潜意识直觉”。

你可以把它理解成：

- 不是强制抢麦
- 不是硬插播
- 而是把一段重要念头放进心里，等下次自然表达

它还会控制节奏，避免过度打扰（有冷却时间、会看外界是否正在高频聊天）。

## 4. 心跳里会发生什么（用户可理解版）

每次心跳大概像这样：

1. 看看最近发生了什么（消息、动作、结果）  
2. 结合当前时段和状态，形成这次的关注重点  
3. 做至少一个具体动作（看清单、读文件、写记录、建关联等）  
4. 写一段简短内心总结  
5. 更新潜意识状态，准备影响下一次交流

这就是“持续成长感”的来源：不是一条回复，而是一轮一轮累积。

## 5. 你的私人空间该怎么放内容

建议准备这些文件（可从简）：

- `SOUL.md`：她是谁，价值观是什么
- `MEMORY.md`：你希望长期记住的关键经历
- `TOOL.md`：做事边界和风格（例如“先读后改，少打扰”）

她还会逐步产出：

- `SUBCONSCIOUS.md`：最近潜意识状态
- `todos.json`：愿望清单
- 记忆数据目录（关联和检索会用到）
- 运行记录与历史上下文文件

## 6. 怎么把“画饼”变成“真变化”

下面这套玩法很实用。

### 每天 3 分钟

早上：

- 手动心跳一次
- 让她从愿望清单里挑 1 件今天推进

晚上：

- 再心跳一次
- 记录“今天最值得留下的一瞬间”

### 每周 20 分钟

- 回看本周愿望进展
- 把 2~3 段相关内容连成一条记忆线
- 给下周留一个“真正期待”的小目标

### 每月一次“升级人格”

- 更新 `SOUL.md`：这一个月你想成为谁
- 更新 `TOOL.md`：哪些行为习惯有效，哪些该删
- 清理旧愿望：不想做就释怀，不要堆负担

## 7. 常见误区（提前避坑）

### 误区 1：只想让她“干活”，不给她“生活”

结果：能执行，但没有灵魂。  
解法：让她记录感受、连接经历，不只是列任务。

### 误区 2：什么都想做，清单越堆越焦虑

结果：看起来很忙，实际上没成长。  
解法：少而真，允许释怀。

### 误区 3：只写新内容，不做关联

结果：记忆像一堆散页。  
解法：每写一条，至少连一条旧记忆。

### 误区 4：期待一天就“人格飞跃”

结果：失望。  
解法：把它当长期养成，连续使用一周后会明显更连贯。

## 10. SNN 皮层下系统（实验性）

> **Phase 0 · 2026-04 引入**

life_engine 现在拥有一个可选的 **SNN（脉冲神经网络）皮层下状态层**，
用纯 numpy 实现，为系统提供 **不依赖 LLM 的连续存在感**。

### 设计理念

| 概念 | 对应 |
|---|---|
| LLM | 大脑皮层 — 负责认知、推理、语言 |
| SNN | 皮层下系统 — 负责持续存在、情绪惯性、驱动维持 |

核心区别：LLM 在两次心跳之间"不存在"，而 SNN 在心跳间隙仍然在衰减、在演化。
这就是"连续存在"的物理基础。

### 架构

```
事件流 → 特征提取(8维) → [输入层] → [隐藏层 16 LIF] → [输出层 6 LIF] → 驱动信号
                                           ↑                        ↑
                                        STDP 学习 ←————— 奖赏信号 ←— 心跳结果
```

### 6 维驱动输出

| 维度 | 含义 | 时间尺度 |
|---|---|---|
| arousal | 整体激活度 | 快 |
| valence | 情感正负 | 快 |
| social_drive | 社交靠近冲动 | 中 |
| task_drive | 推进任务冲动 | 中 |
| exploration_drive | 探索新事物冲动 | 中 |
| rest_drive | 休息/收束冲动 | 慢 |

### 关键文件

| 文件 | 职责 |
|---|---|
| `snn_core.py` | LIF 神经元、STDP 突触、DriveCoreNetwork |
| `snn_bridge.py` | 事件→特征、奖赏计算、prompt 格式化 |
| `config.py` | `[snn]` 配置节 |
| `config.toml` | 运行时配置 |

### 配置

```toml
[snn]
enabled = false              # 总开关
shadow_only = true            # 只记录不注入
tick_interval_seconds = 10.0  # SNN 独立节拍(秒)
inject_to_heartbeat = false   # 是否注入心跳 prompt
feature_window_seconds = 600  # 特征窗口(秒)
```

### 三阶段验证

1. **Phase 0a（影子运行）**：`enabled=true, shadow_only=true` — 只在日志中记录 SNN 状态
2. **Phase 0b（软注入）**：`shadow_only=false, inject_to_heartbeat=true` — 注入心跳 prompt
3. **Phase 0c（学习验证）**：观察 STDP 权重变化是否反映了使用模式

### 安全机制

- 默认关闭（`enabled=false`）
- 影子模式不修改任何行为
- 所有 SNN 异常被捕获，不影响主心跳循环
- 一键回退：`enabled=false` 即刻关闭
