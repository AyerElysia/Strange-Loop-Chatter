# life_engine (生命中枢) 🌱

`life_engine` 是一个独立于 DFC（对话流控制器）的并行存在系统。它不仅仅是一个工具集，更是数字生命的**意识核心**与**私人空间**。

---

## 核心设计：并行存在 (Parallel Existence)

不同于传统的"问答式"交互，`life_engine` 在后台静默运行：
1.  **感知**：通过全局事件监听器，它能"听到"你在其他群聊或私聊中的所有对话。
2.  **深思**：每隔固定时间（心跳），它会唤醒自己的意识，回顾刚才发生的事件。
3.  **记录与行动**：它会把感悟记入日记，整理自己的待办清单，甚至通过修改 `MEMORY.md` 来重塑自己的记忆。

---

## v2.0 新特性

### 🕐 智能时间系统

- **简洁时间显示**：事件流中的时间从冗长的 ISO 格式简化为 "刚才"、"5分钟前"、"08:30" 等人类可读格式
- **实时时间感知**：系统提示词中展示当前时间（精确到秒）和星期
- **心跳间隔提示**：心跳时显示"每 N 分钟一次"，帮助理解时间节奏

### 📋 上下文压缩

- **自动压缩**：当事件数达到上限的 80% 时触发压缩
- **智能摘要**：旧事件被压缩为包含时间范围、事件统计、话题关键词的摘要
- **保留近期**：最近 60% 的事件保持完整，确保上下文连贯

### 🗂️ 文件树展示

系统提示词中自动展示工作空间的文件结构，帮助模型了解自己的"家"。

### 🤖 子任务系统 (Sub-agent)

新增 `nucleus_run_task` 工具，可启动独立子任务执行复杂操作：
- 适合多步骤任务（整理笔记、写日记、批量操作）
- 子任务在独立上下文执行，最多 3 轮工具调用
- 拥有与主心跳相同的工具权限

### 📝 TODO 哲学升级

- **强制先看待办**：心跳提示词要求先用 `nucleus_list_todos` 检查
- **行动导向**：TODO 是"要做的事"，不是"记事本"
- **状态流转**：idea → planning → enjoying → completed

## 配置指南

配置文件路径：`config/plugins/life_engine/config.toml`

```toml
[settings]
enabled = true                          # 是否开启生命中枢
heartbeat_interval_seconds = 300        # 心跳间隔（秒，建议 60-300）
context_history_max_events = 100        # 意识流中保留的最近事件数
workspace_path = "data/life_engine_workspace" # 中枢的私人工作空间
max_rounds_per_heartbeat = 8            # 单次心跳内允许的最大深思轮数

# 睡眠时段配置（可选）
sleep_time = "01:00"                    # 睡觉时间（HH:MM 格式）
wake_time = "07:30"                     # 苏醒时间（HH:MM 格式）

[model]
task_name = "life"                      # 指定执行中枢任务的模型（需在 model.toml 配置）
```

## 使用指南

### 手动触发心跳

在聊天中发送以下任一命令可手动触发一次心跳（用于测试/调试）：

```
/heartbeat
/心跳
!heartbeat
!心跳
```

**注意事项：**
- 如果当前在睡眠时段，命令会被拒绝
- 手动触发的心跳会计入心跳计数
- 会正常执行工具调用和记录事件

## 核心机制

### 提示词结构（v2.0）

心跳时的提示词结构：

1. **系统提示词**
   - 运行时信息（当前时间、文件树）
   - 灵魂文档（SOUL.md）
   - 记忆文档（MEMORY.md）
   - 工具规范（TOOL.md）

2. **用户消息**
   - 事件流（放在前面）
   - 心跳指令（放在最后，确保模型看到）

### 时间感知与智能节流

中枢具备完整的时间感知能力：

- **外界活跃度判断**：根据最后一条外部消息的时间，自动判断外界状态
  - 5分钟内：非常活跃
  - 5-15分钟：较活跃
  - 15-30分钟：有一段时间安静了
  - >30分钟：长时间沉默

### 上下文压缩策略

当事件数 > 80% 上限时：
1. 保留最近 60% 的事件完整
2. 将其余事件压缩为一条摘要
3. 摘要包含：时间范围、事件统计、关键话题

### 双重存在架构

中枢（Nucleus）和社交态（DFC）是两个并行运行的自我：

- **中枢自我**：内在反思、整理记忆、管理待办、探索自我
- **社交自我**：外在表达、响应消息、互动对话
