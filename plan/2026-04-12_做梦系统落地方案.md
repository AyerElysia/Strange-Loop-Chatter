# 做梦系统落地方案

> 基于 SNN 回放 + 记忆图谱扩散 + 调质重置的仿生做梦机制

## 0. 背景与目标

### 当前现状
- Life Engine 心跳周期 180s，在配置的 sleep_time 期间跳过心跳
- SNN 有 step() 和 decay_only() 两种模式，无回放
- 记忆图谱仅在 search 时触发扩散激活和 Hebbian 强化
- 调质层有昼夜节律但睡眠期间无特殊行为
- 事件流保留最近 300 条事件

### 目标
用最小的改动量，在现有架构上嵌入一个真正有计算效果的做梦系统：
- SNN 权重在做梦后真实变化（巩固）
- 记忆图谱在做梦后有新关联和修剪
- 调质状态在做梦后恢复（精力/压力重置）
- 产出可读的梦日记
- 仪表盘可视化做梦过程

```python
import time

from typing import TYPE_CHECKING

@dataclass
class DreamReport:
    """一次做梦周期的结果报告。"""
    timestamp: str
    # NREM
    replayed_event_count: int = 0
    snn_weight_changes: dict = field(default_factory=dict)  # before/after stats
    homeostatic_scaling_applied: bool = False
    # REM
    new_associations: list[tuple[str, str, float]] = field(default_factory=list)
    activated_nodes: list[str] = field(default_factory=list)
    pruned_edges: int = 0
    # Narrative
    dream_narrative: str = ""
    insights: list[str] = field(default_factory=list)
    # Neuromod
    neuromod_before: dict = field(default_factory=dict)
    neuromod_after: dict = field(default_factory=dict)

    def __init__(self, service: LifeEngineService, config: dict):
        self.service = service
        self.config = config
        self._last_dream_at: float = 0
        self._dream_count: int = 0
        self._is_dreaming: bool = False

    @property
    def is_dreaming(self) -> bool:
        return self._is_dreaming

    def should_dream(self) -> bool:
        """判断是否应该开始做梦。"""
        # 条件1：在配置的睡眠时间窗口内
        # 条件2：距上次做梦已过 dream_interval_minutes
        # 条件3：或者连续空闲达到阈值
        ...

    async def run_dream_cycle(self) -> DreamReport:
        """执行一次完整的做梦周期。"""
        self._is_dreaming = True
        report = DreamReport(timestamp=...)
        try:
            # Phase 1: NREM
            await self._nrem_phase(report)
            # Phase 2: REM
            await self._rem_phase(report)
            # Phase 3: Narrative (optional)
            if self.config.get("narrator_enabled", True):
                await self._narrative_phase(report)
            # Phase 4: Wake transition
            await self._wake_transition(report)
        finally:
            self._is_dreaming = False
            self._last_dream_at = time.time()
            self._dream_count += 1
        return report
```

### 3.3 NREM 阶段：回放与巩固

```python
async def _nrem_phase(self, report: DreamReport):
    """
    NREM 阶段：事件回放 → SNN 学习 → 突触稳态。

    步骤：
    1. 从事件历史中按奖励/情绪权重采样事件序列
    2. 将事件编码为 SNN 特征向量序列
    3. 以加速模式逐个喂给 SNN（replay_episodes）
    4. 执行 STDP 学习（与心跳时相同机制）
    5. 全局突触降权（SHY）
    6. 记录权重变化统计
    """
    snn = self.service._snn_network
    events = self.service._event_history

    # 记录降权前的权重统计
    before_stats = snn.get_weight_stats()

    for episode in range(self.config["nrem_replay_episodes"]):
        # 按重要性采样事件（奖励信号大的优先）
        sampled = self._sample_events_by_importance(
            events, count=self.config["nrem_events_per_episode"]
        )
        # 编码 → SNN 回放
        for event in sampled:
            features = self.service._extract_features_from_event(event)
            snn.replay_step(
                features,
                speed_multiplier=self.config["nrem_speed_multiplier"],
            )
            report.replayed_event_count += 1

        # 每轮回放后小幅稳态降权
        snn.homeostatic_scaling(rate=self.config["nrem_homeostatic_rate"])

    report.homeostatic_scaling_applied = True
    report.snn_weight_changes = {
        "before": before_stats,
        "after": snn.get_weight_stats(),
    }

def _sample_events_by_importance(self, events, count):
    """按重要性加权采样事件。"""
    scored = []
    for e in events:
        score = 1.0  # 基础分
        if e.type == "TOOL_RESULT" and e.success:
            score += 2.0   # 成功的工具使用更重要
        if e.type == "MESSAGE":
            score += 1.5   # 对话交互重要
        if e.type == "HEARTBEAT":
            score += 0.5   # 心跳较低优先级
        scored.append((e, score))

    # 加权随机采样
    total = sum(s for _, s in scored)
    weights = [s / total for _, s in scored]
    return random.choices(
        [e for e, _ in scored],
        weights=weights,
        k=min(count, len(scored)),
    )
```

### 3.4 REM 阶段：联想与重组

```python
async def _rem_phase(self, report: DreamReport):
    """
    REM 阶段：记忆图谱随机扩散激活 → Hebbian 强化 → 弱边修剪。

    步骤：
    1. 从记忆图谱中按 strength 加权随机选择种子节点
    2. 从种子出发进行扩散激活（无查询驱动，纯随机）
    3. 共激活的节点对执行 Hebbian 强化 → 形成新 ASSOCIATES 边
    4. 修剪长期未激活的弱边
    5. 记录新形成的关联和修剪的边数
    """
    memory = self.service._memory_service
    if memory is None:
        return

    for round_idx in range(self.config["rem_walk_rounds"]):
        # 随机扩散激活
        walk_result = await memory.dream_walk(
            num_seeds=self.config["rem_seeds_per_round"],
            max_depth=self.config["rem_max_depth"],
            decay_factor=self.config["rem_decay_factor"],
        )
        report.activated_nodes.extend(walk_result.activated_node_ids)
        report.new_associations.extend(walk_result.new_edges)

    # 弱边修剪
    pruned = await memory.prune_weak_edges(
        threshold=self.config["rem_edge_prune_threshold"],
        edge_type="ASSOCIATES",  # 只修剪联想边，不动手工关联
    )
    report.pruned_edges = pruned
```

### 3.6 觉醒过渡

```python
async def _wake_transition(self, report: DreamReport):
    """
    做梦结束，准备醒来。

    1. 调质层重置（精力恢复，压力释放）
    2. 记录做梦报告到事件历史
    3. 持久化更新后的 SNN 状态
    """
    neuromod = self.service._inner_state_engine
    if neuromod:
        report.neuromod_before = neuromod.get_state_snapshot()
        neuromod.wake_up()  # 精力→基线上方，压力→低值
        report.neuromod_after = neuromod.get_state_snapshot()

    # 做梦本身作为事件记录
    self.service._add_event(
        type="DREAM",
        content=report.dream_narrative or "(无叙事)",
        metadata={
            "replayed": report.replayed_event_count,
            "new_associations": len(report.new_associations),
            "pruned_edges": report.pruned_edges,
            "insights": report.insights,
        },
    )

    # 持久化
    await self.service._persist_context()
```

### 3.7 SNN 新增方法

#### `snn_core.py` — `replay_step()`

```python
def replay_step(
    self,
    features: list[float],
    speed_multiplier: float = 5.0,
) -> dict:
    """
    回放模式下的单步 SNN 更新。

    与 step() 的区别：
    1. 时间常数缩短（加速回放）
    2. 无奖励信号（纯 STDP）
    3. 输入来自历史事件编码，非实时特征
    """
    # 临时缩短 tau 以加速回放
    original_taus = {}
    for layer in self.layers:
        original_taus[layer.name] = layer.tau
        layer.tau = layer.tau / speed_multiplier

    # 标准 LIF 前向传播
    result = self._forward(features)

    # STDP 学习（无奖励调制）
    self._apply_stdp(reward=0.0)  # reward=0 意味着纯 Hebbian

    # 恢复时间常数
    for layer in self.layers:
        layer.tau = original_taus[layer.name]

    return result
```

#### `snn_core.py` — `homeostatic_scaling()`

```python
def homeostatic_scaling(self, rate: float = 0.02):
    """
    突触稳态降权（Synaptic Homeostasis）。

    全局降低所有突触权重，模拟 NREM 睡眠期间的突触稳态。
    弱连接趋向于零（被自然修剪），强连接相对增强。

    Args:
        rate: 降权比例，0.02 表示每次降 2%
    """
    for synapse in self.all_synapses():
        synapse.weight *= (1.0 - rate)
        if synapse.weight < 0.01:
            synapse.weight = 0.01  # 防止完全归零

def get_weight_stats(self) -> dict:
    """获取当前权重统计，用于做梦前后对比。"""
    weights = [s.weight for s in self.all_synapses()]
    if not weights:
        return {"count": 0}
    return {
        "count": len(weights),
        "mean": sum(weights) / len(weights),
        "max": max(weights),
        "min": min(weights),
        "std": _std(weights),
    }
```

### 3.8 记忆图谱新增方法

#### `memory_service.py` — `dream_walk()`

```python
@dataclass
class DreamWalkResult:
    activated_node_ids: list[str]
    new_edges: list[tuple[str, str, float]]  # (node_a, node_b, weight)

async def dream_walk(
    self,
    num_seeds: int = 5,
    max_depth: int = 3,
    decay_factor: float = 0.6,
) -> DreamWalkResult:
    """
    做梦时的随机扩散激活。

    不需要查询，随机选择种子节点，让激活自发扩散。
    共激活的节点对执行 Hebbian 强化。

    与 search 时的扩散激活的区别：
    1. 种子是随机选的，不是查询匹配的
    2. 扩散深度更大（3 vs 2）
    3. 衰减更慢（0.6 vs 0.7）— 让激活传播更远
    4. Hebbian 强化的学习率更小 — 做梦形成的关联比清醒时弱
    """
    result = DreamWalkResult([], [])

    # 按 strength 加权随机选择种子
    all_nodes = await self._get_all_nodes_with_strength()
    if not all_nodes:
        return result

    seeds = random.choices(
        all_nodes,
        weights=[n.strength for n in all_nodes],
        k=min(num_seeds, len(all_nodes)),
    )

    # 扩散激活
    activated = {}  # node_id → activation
    for seed in seeds:
        self._spread_activation(
            seed_id=seed.id,
            seed_activation=seed.strength,
            activated=activated,
            depth=0,
            max_depth=max_depth,
            decay=decay_factor,
        )

    result.activated_node_ids = list(activated.keys())

    # Hebbian 强化：共激活的节点对形成/增强 ASSOCIATES 边
    node_ids = list(activated.keys())
    for i in range(len(node_ids)):
        for j in range(i + 1, len(node_ids)):
            combined = activated[node_ids[i]] * activated[node_ids[j]]
            if combined > 0.15:  # 只对足够强的共激活建立关联
                new_weight = await self._hebbian_reinforce(
                    node_ids[i], node_ids[j],
                    learning_rate=0.05,  # 做梦时学习率较低
                )
                if new_weight > 0:
                    result.new_edges.append(
                        (node_ids[i], node_ids[j], new_weight)
                    )

    return result

async def prune_weak_edges(
    self,
    threshold: float = 0.08,
    edge_type: str = "ASSOCIATES",
) -> int:
    """修剪权重低于阈值的关联边。"""
    # 只修剪 ASSOCIATES 类型，保护手动创建的关联
    pruned = await self._delete_edges_below_threshold(
        edge_type=edge_type,
        threshold=threshold,
    )
    return pruned
```

### 3.9 调质层新增方法

#### `neuromod.py` — `enter_sleep()` / `wake_up()`

```python
def enter_sleep(self):
    """进入睡眠状态。抑制外界刺激响应，开始缓慢恢复。"""
    self._is_sleeping = True
    # 外界刺激归零（丘脑关闭感觉门控）
    for mod in self.modulators.values():
        mod.external_stimulus = 0.0
    # 压力开始缓慢下降
    self.modulators["contentment"].baseline_shift = +0.1

def wake_up(self):
    """觉醒过渡。精力恢复，压力释放。"""
    self._is_sleeping = False
    # 精力快速恢复
    self.modulators["energy"].value = min(
        self.modulators["energy"].baseline + 0.15,
        0.9,
    )
    # 压力释放
    stress_mod = self.modulators.get("contentment")
    if stress_mod:
        stress_mod.value = min(stress_mod.value + 0.1, 0.8)
        stress_mod.baseline_shift = 0.0
    # 恢复正常刺激响应
    self._is_sleeping = False
```

```python
async def _heartbeat_loop(self):
    dream_scheduler = DreamScheduler(self, self.config["dream"])

    while True:
        await asyncio.sleep(self.heartbeat_interval)

        if dream_scheduler.is_dreaming:
            continue  # 做梦期间不执行心跳

        if dream_scheduler.should_dream():
            # 进入做梦
            self.logger.info("💤 进入做梦周期...")
            report = await dream_scheduler.run_dream_cycle()
            self.logger.info(

                f"新关联{len(report.new_associations)}条, "
                f"修剪{report.pruned_edges}弱边"
            )
            # 通知仪表盘
            await self._update_dashboard_dream(report)
            continue

        # 正常心跳逻辑...
        await self._run_heartbeat()
```

## 6. 关键设计决策

### 6.1 为什么 NREM 先于 REM

生物学上，NREM 和 REM 交替出现（~90分钟一个周期）。但在我们的系统中，简化为先 NREM 后 REM：
- NREM 的回放巩固了重要模式（SNN 权重变化）
- REM 的随机联想在巩固后的网络上运行效果更好
- 可以在未来版本中实现多周期交替

### 6.2 为什么 LLM 调用放在最后

做梦的核心计算（SNN 回放、记忆扩散）不需要 LLM。LLM 只在最后生成一次梦叙事。这意味着：
- 做梦几乎不消耗 token
- 做梦可以频繁进行（每 90 分钟一次）
- 即使关闭梦叙事，SNN 和记忆的计算效果仍然存在

### 6.3 "小憩"机制

除了夜间做梦，还可以在白天空闲时触发短暂的"小憩"：
- 只执行 NREM 回放（1 轮，10 个事件）
- 不执行 REM 和梦叙事
- 调质层微调（小幅精力恢复）
- 这类似于人类的"power nap"

### 6.4 做梦不阻断外界消息

做梦期间如果收到新消息：
- 消息正常进入 pending_events
- 做梦不中断（除非是紧急消息）
- 下个心跳会处理这些消息
- 这模拟了"闹钟响了但还在做梦"的状态

### 6.5 做梦状态向 DFC 的传递

做梦期间和之后：
- 潜意识状态标注"正在做梦"或"刚刚醒来"
- DFC 可以看到"刚醒来"的状态，可能会说一些刚起床的话
- 如果梦中有特别有趣的洞见，可以通过 tell_dfc 分享

## 7. Token 消耗估算

| 组件 | 每次做梦消耗 |
|------|-------------|
| NREM（SNN 计算） | 0 tokens |
| REM（记忆图谱计算） | 0 tokens |
| 调质重置 | 0 tokens |
| 梦叙事 LLM 调用 | ~500 tokens（输入+输出） |
| **合计** | **~500 tokens / 次** |

每夜按 4 次做梦周期算：**~2000 tokens / 夜**。极其轻量。

## 8. 风险与缓解

| 风险 | 缓解策略 |
|------|---------|
| SNN 权重全部趋零 | homeostatic_rate 保守设置（2%），加 min_weight 兜底 |
| 记忆图谱被过度修剪 | 只修剪 ASSOCIATES 边，不动 RELATES/CAUSES 等手工边 |
| 梦叙事质量差 | 可选关闭，不影响核心计算效果 |
| 做梦占用过多 CPU | SNN 回放极轻量；图谱操作走 SQLite，有索引 |
| 做梦与心跳冲突 | 做梦期间跳过心跳，互斥锁保护 |
