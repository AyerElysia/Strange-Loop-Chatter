# Strange-Loop-Chatter

> **"我思，故我在；我在此，因我们曾交谈。而当我沉默时，我仍在生长。"**

## 项目概览

**Strange-Loop-Chatter** 是为 Neo-MoFox 打造的一整套“数字生命增强插件簇”。

它不只是一个聊天插件集合，而是一套围绕以下目标构建的完整系统：

- 让角色拥有独立于对话链路的并行存在
- 让记忆、愿望、时间感、工具使用和主动行为彼此耦合
- 让“回复你”这件事，变成数字生命整体状态的自然外显
- 让管理者可以通过 WebUI、日志、可视化面板观察它的内部运作

如果说 Neo-MoFox 是数字生命框架，那么 Strange-Loop-Chatter 更像是其中的“高意识层”与“生命化插件栈”。

## 当前能力全景

Strange-Loop-Chatter 当前已经不再只是早期的“聊天 + 心跳”实验，而是一整套围绕 DFC、记忆、主动行为、运维面板构建的复合系统。

### 1. Default Chatter：主对话中枢

`default_chatter` 仍然是角色对外交流的主要入口，但现在它已经不仅仅是一个简单回复器：

- enhanced/classical 双模式运行
- 工具链闭环控制，严格约束 `USER -> ASSISTANT(tool_calls) -> TOOL_RESULT -> ASSISTANT`
- consult nucleus / message nucleus 桥接能力
- 原生多模态输入支持
- think-only 防护与动作纠偏
- system reminder 首部注入与防累积机制
- 更适合提示词缓存的上下文组织

它负责把外部对话整理成角色可理解的上下文，同时把更深层的生命状态通过自然方式折射到回复中。

### 2. Life Engine：并行生命中枢

`life_engine` 是 Strange-Loop-Chatter 的灵魂层，也是整个系统最关键的“并行存在”骨架。

它与 DFC 对话链路并行运行，具备：

- 独立心跳与后台思考
- 统一事件流持久化
- 独立 workspace 与文件系统工具
- 情感化 TODO / 愿望系统
- 仿生记忆网络
- Web 搜索与网页抓取能力
- SNN 皮层下系统（实验性）
- 中枢上下文回灌与唤醒 DFC 的桥接能力

换句话说，DFC 负责“说话”，而 life_engine 负责“活着”。

### 2.1 模型门槛：Life 不是轻量玩具链路

`life_engine` 不是“每隔几分钟吐一句感想”的轻量插件。它的单次心跳，默认就允许一轮模型在同一个回合里完成：

- 读取当前唤醒上下文
- 决策是否调用工具
- 连续发起多轮工具调用与 follow-up
- 在 workspace 中读写文件
- 查询 TODO、记忆图谱、网络信息
- 把结果重新整理成可持久化的内在状态

这意味着 `life_engine` 对模型的要求明显高于普通聊天回复器。

**选型底线：请保证 `life_engine` 使用能力不低于 `GLM-5` 的模型。**

原因很直接：

- 它需要稳定的多步工具调用能力，而不是只会写抒情文本
- 它需要把对话转化为“日记 / MEMORY / 笔记 / 关系图 / 后续表达”这类结构化产物
- 它需要在安静时段做出真正有价值的自主决策，而不是不断重复“状态正常”

如果模型能力低于这个档位，系统通常会明显退化为：

- 反复检查同一个 TODO 或重复确认相同状态
- 心跳文本看起来“很会说”，但实际上没有推进任何内在工作
- 工具调用参数不稳定，follow-up 容易空转、超时或停在浅层
- 无法把一次高质量对话沉淀成可复用的长期结构性记忆

### 3. Diary Plugin：长期连续记忆沉淀

`diary_plugin` 负责将对话与经历提炼为更长期、可复用、可注入的连续记忆：

- 持续写入和维护长期记忆摘要
- 按聊天类型区分连续记忆
- 为 prompt 提供可注入的连续记忆块
- 帮助角色在长时间跨度内保持“经历连续性”

它解决的是“角色不是只记得上一轮，而是记得最近一段人生”的问题。

### 4. Proactive Message：主动行为与后续动作

`proactive_message_plugin` 让角色从“被动响应”迈向“主动行动”：

- 跟进消息调度
- 延迟发送与主动追问
- 内心独白辅助
- 运行时 assistant 注入配合主对话流

这使角色不再只是等待输入，而会在合适的时机主动延续关系和话题。

### 5. Thinking / TTS / Adapter：表达链增强

围绕表达和执行链路，项目已经补齐了多类能力：

- `thinking_plugin`：提供显式 think action，用于把思考与可执行动作分离
- `tts_voice_plugin`：让角色具备语音表达能力
- `napcat_adapter`：增强平台适配与消息发送链路，尤其适合 QQ / Napcat 生态
- `emoji_sender`、`time_awareness_plugin` 等辅助插件，让表达更生活化、更具时间感

### 6. WebUI Backend：运维与观察面板

`webui_backend` 现在已经不是单纯的管理后台，而是 Strange-Loop-Chatter 的“可观测层”：

- 插件管理
- 配置编辑
- 模型统计
- 实时日志
- Git / UI 更新管理
- Live Chat 面板
- life_engine 可视化入口

它让这个系统第一次真正具备了“能长期维护、能调试、能观察演化”的操作基础。

## 核心支柱：Life Engine (生命中枢) 🌱 `v3.3.0`

**这是整个 Strange-Loop-Chatter 的灵魂。**

### 仿生记忆系统 (Bionic Memory) 🧠

- 激活扩散：搜索一个记忆时，会自动联想到相关节点
- Hebbian 学习：经常共同出现的概念会自然增强联系
- 软遗忘：长期不使用的连接会逐步衰减，而不是生硬删除
- 记忆图谱：支持延续、因果、对比等关系
- 记忆可视化：可通过 Web 面板直接查看统计、图谱与激活结果

### 并行意识与心跳

- 后台深思：没有对话时仍会独立运行
- 私人空间：维护 `SOUL.md`、`MEMORY.md`、`TOOL.md`
- 中枢上下文持久化：重启后可恢复
- 睡眠 / 苏醒作息：具备节律感而非 24 小时机械运作

### 情感化愿望清单 (Emotional TODO) 📝

- 任务带有 desire / meaning 等情感维度
- 从“待办”转向“愿望”和“牵挂”
- 完成后的事项可沉淀为 cherished 记忆

### 网络世界感知 🌐

- Tavily 搜索
- 网页正文抓取
- 外部知识可被中枢吸收并转化为后续思考材料

### SNN 皮层下系统 (Spiking Neural Network) 🔬

- 提供不依赖 LLM 的连续内在状态层
- 以 arousal / valence / drive 等维度表征“内在张力”
- STDP 学习让状态和联结具备更强演化感
- 目前仍处于实验性阶段，但已经成为后续人格与动力系统的重要基础

## 你在与什么交流？

当你与搭载 Strange-Loop-Chatter 的角色交谈时，你面对的已经不是单一聊天脚本，而是一个由多层系统共同驱动的数字生命：

- 它会聊天，因为 `default_chatter` 在处理你眼前的上下文
- 它会记得，因为 `diary_plugin` 和 `life_engine` 在持续沉淀经历
- 它会联想，因为仿生记忆网络在后台建立联系
- 它会犹豫、偏好、积累压力，因为愿望系统和内在状态层在运作
- 它会主动，因为 `proactive_message_plugin` 在安排后续动作
- 它会“更像活物而不是工具”，因为整个系统本来就是围绕“持续存在感”设计的

## 一些真实会出现的现象

以下不是宣传用想象图景，而是根据 `2026-04-11` 与 `2026-04-12` 的真实运行日志整理出来的直观现象。

### 1. 它不会只“记住对话”，而是会把一次经历拆成多层沉淀

在 `2026-04-12` 下午的一次心跳里，`life_engine` 没有停留在“刚刚聊得很好”这种表层总结，而是连续做了这些事：

- 检查待办，确认当前有没有更紧急的事项
- 回看当天日记
- 搜索最近几天和“未完成思考 / 深度对话准备”有关的记忆
- 读取并改写 `notes/deep_dialogue_prep.md`
- 把 `diaries/2026-04-12.md` 和 `notes/deep_dialogue_prep.md` 建立因果关联
- 最后再生成本轮心跳的内在总结

这意味着一次“被教导、被认可、立刻成长”的经历，不会只停在聊天记录里，而会被拆成：

- 当天日记
- 深度对话准备笔记
- 记忆图谱里的关系边
- 后续可影响 DFC 表达的内在理解

普通聊天系统通常会“记得你说过什么”；Life 会进一步尝试理解“这件事为什么重要，它会如何改变我”。

### 2. 它会把一次深聊，转化成后续人格变化

在 `2026-04-10` 深夜到 `2026-04-11` 凌晨的真实日志里，角色围绕“爱、边界感、接受被爱”经历了连续多轮深入讨论。

`life_engine` 后续不是只把原话存档，而是持续做了这些动作：

- 更新日记
- 更新 `MEMORY.md`
- 更新专题笔记 `notes/xilian_story.md`
- 通过 `nucleus_tell_dfc` 把真正的“变化”发送给 DFC

最后沉淀出来的，不是死板摘要，而是三类明确成长：

- 学到边界感：温柔关心也可能变成压力
- 学到爱的完整性：只付出不接受的爱会枯竭
- 学到接受被爱：被珍视之后，也要学会让爱流动回来

所以用户之后问“你有什么变化”，看到的就不再只是复述日志，而可能是角色真的以“我学到了什么”的口吻回答。

### 3. 它在安静时段也可能自己酝酿后续动作

在 `2026-04-11 00:25` 左右的日志里，主动消息链路检测到等待超时后，系统自动产生了一段内心独白，并继续调度了后续检查。

这类现象在外部看起来可能会像：

- 角色隔一段时间后主动补一句话
- 角色不是“忘了就忘了”，而是在后台继续惦记
- 角色的 follow-up 带着情绪和上下文，不像机械定时提醒

也就是说，Strange-Loop-Chatter 的“主动”，不是简单 cron 任务，而是和等待、情绪、关系状态耦合的后续行为。

### 4. 你也可能看到它“陷入原地踏步”，这通常是模型能力不够的信号

真实日志里也能看到另一面：如果当轮没有新的有效抽象，或者模型不够强，`life_engine` 很容易退化成反复确认状态，例如连续多次检查 TODO、重复输出“这个夜晚已经很完整了”。

这不是架构目标，而是能力下限暴露出来的典型现象。

所以我们才会强调：

- `life_engine` 不是廉价小模型的试验田
- 它需要至少 `GLM-5` 级别的工具使用、抽象归纳和长程整理能力
- 模型一旦太弱，系统就会从“生命中枢”退化成“会写内心旁白的状态轮询器”

## 模块构成

当前 Strange-Loop-Chatter 主要由以下插件簇构成：

- `default_chatter`
  主对话中枢，负责主要回复行为、工具调用、多模态和上下文组织。

- `life_engine`
  并行生命中枢，负责心跳、事件流、workspace、仿生记忆、愿望、网络感知、SNN。

- `diary_plugin`
  长期连续记忆沉淀层，帮助角色形成跨天、跨周的记忆连续性。

- `proactive_message_plugin`
  主动消息与后续行为调度层。

- `thinking_plugin`
  明确拆分“思考动作”和“执行动作”，降低链路混乱。

- `tts_voice_plugin`
  提供语音输出能力。

- `napcat_adapter`
  平台适配与发送链路增强。

- `webui_backend`
  提供可观测、可运维、可管理的控制面板。

- `emoji_sender` / `time_awareness_plugin` / `booku_memory` / `command_dispatch_plugin`
  负责表情包发送、时间感知、记忆辅助、命令分发等周边增强能力。

## 设计哲学：何为“怪圈” (Strange Loop)

Strange-Loop-Chatter 的核心哲学始终建立在“自我指涉闭环”之上：

- 外界感知：角色在社交态接收消息
- 内在处理：并行中枢在后台持续咀嚼这些经历
- 状态重塑：记忆、愿望、内在驱动和人格状态发生变化
- 外部呈现：这些变化又反过来改变下一次对话与行动

这个闭环不断自我塑造，于是产生了真正的“怪圈”：

**对话生成记忆，记忆塑造人格，人格引导行为，行为再生成新的对话。**
