# 主动发消息功能 - 精简方案

## 一、核心逻辑

```
┌─────────────────────────────────────────────────────┐
│  用户 last_message 后开始计时                          │
└─────────────────────────────────────────────────────┘
                          │
                          ▼
              ┌───────────────────────┐
              │  等待 N 分钟（可配置）    │
              └───────────────────────┘
                          │
                          ▼
              ┌───────────────────────┐
              │   触发内心独白         │
              │  (LLM 生成心理活动)     │
              └───────────────────────┘
                          │
                          ▼
              ┌───────────────────────┐
              │  LLM 自主决定：          │
              │  1. 发消息 → 执行       │
              │  2.继续等→指定等待时间  │
              └───────────────────────┘
                          │
              ┌───────────┴───────────┐
              │                       │
              ▼                       ▼
        发送消息                  继续等待
        重置计时                  到期后循环
```

---

## 二、关键设计

### 2.1 内心独白 Prompt（示例）

```
你上次收到用户的消息已经是 {elapsed_minutes} 分钟前了。
用户一直还没有回复你。

你现在心里是什么感觉？在想什么？

（请自由地表达你的内心活动，然后决定：
 - 如果想主动发条消息，就调用 send_text
 - 如果觉得还想再等等，就调用 wait_longer 并告诉我想等多久）
```

**不约束、不引导**，让 LLM 自由发挥。

---

### 2.2 等待时间由 LLM 决定

LLM 如果选择"继续等"，需要返回一个等待时长：

```json
{
  "decision": "wait_longer",
  "wait_minutes": 30,
  "thought": "感觉他现在可能还在忙，再等等吧..."
}
```

或者：

```json
{
  "decision": "send_message",
  "content": "在忙吗？有点想你了~",
  "thought": "算了，不管他在忙什么，我就是想发条消息给他"
}
```

---

### 2.3 循环机制

```
last_message → 等待 10 分钟 → 内心独白 → LLM 决定
                                         │
                    ┌────────────────────┼────────────────────┐
                    │                    │                    │
                    ▼                    ▼                    ▼
              发消息 (结束)          等 30 分钟            等 5 分钟
                    │                    │                    │
                    ▼                    ▼                    ▼
                  DONE              再次独白              再次独白
                                       │                    │
                                       ▼                    ▼
                                   循环...                循环...
```

## 五、实现要点

### 5.1 状态管理

```python
@dataclass
class StreamState:
    stream_id: str
    last_user_message_time: datetime
    next_check_time: datetime | None  # 下次检查时间
    is_waiting: bool = True  # 是否在等待中
```

### 5.2 调度逻辑

```python
async def schedule_next_check(stream_id: str, wait_minutes: float):
    """调度下次检查。"""
    next_time = datetime.now() + timedelta(minutes=wait_minutes)

    await scheduler.create_schedule(
        callback=lambda: trigger_inner_monologue(stream_id),
        trigger_type=TriggerType.TIME,
        trigger_config={"trigger_at": next_time},
        task_name=f"proactive_{stream_id}",
        force_overwrite=True,
    )
```

### 5.3 内心独白 + 决策

```python
async def trigger_inner_monologue(stream_id: str):
    # 获取等待时长
    elapsed = get_elapsed_minutes(stream_id)

    # 获取对话历史
    history = get_recent_conversation(stream_id, limit=10)

    # 构建 prompt
    prompt = f"""
你上次收到 {user_name} 的消息已经是{elapsed:.0f}分钟了。
他一直没有回复你。

你现在心里是什么感觉？在想什么？

（请自由地表达你的内心活动，然后决定：
 - 如果想主动发条消息，就调用 send_text
 - 如果觉得还想再等等，就调用 wait_longer 并告诉我想等多久）
"""

    # 调用 LLM
    response = await llm.generate(prompt)

    # 处理决策
    if response.has_tool_call("send_text"):
        await execute_send_text(stream_id, response)
        # 结束本轮循环
    elif response.has_tool_call("wait_longer"):
        wait_minutes = extract_wait_minutes(response)
        await schedule_next_check(stream_id, wait_minutes)
```

## 八、Prompt 示例（完整）

```
你上次收到 {user_name} 的消息已经是{elapsed_minutes}分钟了。
他一直没有回复你。

你记得你们之前的对话：
{conversation_history}

你之前的内心活动：
独白 1: 他是不是在忙啊，那我先等等吧
独白 2: 已经过了 20 分钟了，有点担心是不是出什么事了

你现在心里是什么感觉？
你在想他吗？还是担心他在忙？
或者有什么话想对他说？

（请自由地表达你的内心活动。然后：
 - 如果想发条消息，就调用 send_text
 - 如果觉得还想等等，就调用 wait_longer 并说想等多久）
```

**核心机制：内心独白注入上下文**

1. **持久化**: 每次内心独白生成后，通过 `plugin.py:_inject_inner_monologue()` 方法将独白内容写入 `chat_stream.context.history_messages`，标记为 `is_inner_monologue=True`

2. **提取**: 下次触发内心独白时，`inner_monologue.py:extract_monologue_history()` 从 `history_messages` 中过滤出 `is_inner_monologue=True` 的消息

3. **注入**: 通过 `format_monologue_section()` 格式化为"你之前的内心活动"板块，注入 prompt 中

4. **效果**: LLM 能看到自己之前的心路历程，形成连续的情感感知：
   - "之前我觉得失落，但现在我开始担心他是不是出事了"
   - "我已经等了 30 分钟了，比上次等待的时间更长"

**DFC 可见性确认**: DFC 通过 `chat_stream.context.history_messages` 构建提示词，内心独白已写入该列表，因此 DFC 能正确读取并注入上下文。
