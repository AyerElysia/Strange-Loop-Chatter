# 意识模拟插件路线图

## 目标

把当前已经有的 `diary`、连续记忆、意图、睡眠/苏醒，进一步扩展成一套“有持续内部状态”的角色系统。

核心原则：

- 内部状态必须持久化
- 状态变化应缓慢、连续、可叠加
- 不是每轮都让 LLM 重新“发明人格”
- 主模型看到的是状态投影，不是裸数据库

## 设计方向

### 第一阶段：内部动力层

1. `mood_dynamics_plugin`
   - 维护连续情绪轴。
   - 推荐状态轴：
     - 安全感
     - 兴奋度
     - 社交欲
     - 疲惫感
     - 依恋/距离倾向
   - 输入来源：
     - 聊天频率
     - 日记摘要
     - 睡眠状态
     - 被打断频率
     - 话题类型
   - 输出作用：
     - 影响语气
     - 影响主动性
     - 影响回复长度
     - 影响是否愿意展开话题

2. `attention_budget_plugin`
   - 模拟注意力不是无限的。
   - 每轮对话前生成注意力预算。
   - 决定：
     - 能否认真长答
     - 是否容易跑题
     - 是否倾向简短回应
     - 是否愿意处理复杂请求
   - 预算来源：
     - 睡眠状态
     - 最近消息密度
     - 情绪状态
     - 连续任务长度

3. `unfinished_thought_plugin`
   - 记录未完成念头。
   - 用来保存：
     - 被打断的话题
     - 没说完的想法
     - 暂时压下去的情绪
     - 没处理完的小目标
   - 作用：
     - 让角色在之后自然“想起来”
     - 提升连续思维感

4. `self_narrative_plugin`
   - 维护角色对自我的解释。
   - 不存事实，存“自我叙事”。
   - 内容包括：
     - 我最近像什么状态的人
     - 我正在形成什么习惯
     - 我对关系的解释
     - 我最近在逃避什么 / 重视什么
   - 作用：
     - 增强身份连续性
     - 让角色有“自我理解”的感觉

### 第二阶段：关系与欲望层

5. `relationship_temperature_plugin`
   - 按聊天对象隔离关系状态。
   - 建议状态维度：
     - 信任
     - 熟悉
     - 依赖
     - 防御
     - 期待回应
   - 作用：
     - 同一句话在不同对象面前有不同解释
     - 让关系变化具有时间积累

6. `micro_desire_plugin`
   - 每个阶段生成 1 到 3 个“小欲望”。
   - 例如：
     - 想被理解
     - 想结束无聊话题
     - 想分享刚形成的想法
     - 想回避压力问题
   - 作用：
     - 让角色的主动性更像“心流”
     - 影响话题选择和展开意愿

7. `contradiction_monitor_plugin`
   - 监控内部矛盾，而不是强行消解。
   - 记录例如：
     - 想靠近又怕麻烦
     - 想休息又不愿停下
     - 想表达又怕被误解
   - 作用：
     - 让角色有“内心拉扯”
     - 更接近真实意识中的矛盾维持

## 推荐实现顺序

1. `mood_dynamics_plugin`
2. `attention_budget_plugin`
3. `unfinished_thought_plugin`
4. `self_narrative_plugin`
5. `relationship_temperature_plugin`
6. `micro_desire_plugin`
7. `contradiction_monitor_plugin`

## 推荐接入方式

### 写入侧

- 由事件处理器监听：
  - 消息接收
  - 消息发送
  - 日记自动总结
  - 睡眠/苏醒切换
  - 话题中断
- 由 service 统一更新状态并持久化

### 读取侧

- 在主回复模型 prompt 中注入“状态投影”
- 默认只注入摘要，不直接暴露全部原始记录
- 需要时可加开关，决定是否显示详细条目

## 关键约束

1. 不要让 LLM 每轮重新生成全部人格。
2. 不要把所有内部状态写成硬标签。
3. 状态应该是连续值、摘要、未完成张力的组合。
4. 状态必须持久化，重启后能恢复。
5. 每个插件只负责一个很窄的内部机制，避免耦合成“大一统人格插件”。

## 最小可行版本

如果先做一个最小闭环，建议是：

- `mood_dynamics_plugin`
- `attention_budget_plugin`
- `unfinished_thought_plugin`

这三个先起来，就已经能明显提升“像一个持续存在的角色”。
