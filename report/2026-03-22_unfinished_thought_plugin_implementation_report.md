# unfinished_thought_plugin 实现报告

## 结论

`unfinished_thought_plugin` 已完成落地实现，当前版本已经具备按聊天流隔离的未完成念头维护能力，并且可以在重启后从磁盘恢复。

## 已实现内容

- 每个私聊/群聊使用独立状态文件持久化存储。
- 每隔固定对话数触发一次自动扫描。
- 扫描时会把当前聊天流的未完成念头状态与最近历史交给扫描模型处理。
- 扫描结果支持增删改和状态更新。
- 主模型 prompt 注入时，会随机抽取 1 到 3 条当前活跃念头。
- 提供了 `/unfinished_thought` 命令集，支持查看、添加、扫描、暂停、恢复和清空。

## 关键落地细节

- 扫描模型默认使用 `diary` 任务，失败时回退到 `actor`。
- 自动扫描失败不会吞掉触发计数，会恢复触发前的消息计数，避免下一轮节奏丢失。
- `add` 命令现在支持直接输入完整文本，带空格内容不会被截断。
- 如果聊天流对象缺少 `context`，扫描过程会安全退化为“空历史”，不会直接崩溃。

## 方案一致性

- 保留了原有自动写日记逻辑，不影响 `diary_plugin` 的日常写入。
- 新插件只负责“未完成念头”池，不替代日记，也不替代自我叙事。
- 思路上遵循“固定扫描 + 随机注入 + 持久化恢复”的简化落地方案，没有引入相关性打分或语义检索。

## 验证结果

- `python -m py_compile` 通过。
- `pytest -q -o addopts='' test/plugins/unfinished_thought_plugin/test_unfinished_thought_plugin.py` 通过。
- 相关自我叙事测试也已通过，说明新插件实现没有破坏同类插件的基础链路。

## 后续建议

- 如果后面要继续增强“意识感”，可以在这个插件之上再加情绪底噪、未完成愿望池和自我叙事联动。
- 如果后续想降低扫描开销，可以再加一个简单的节流窗口，但第一版没有必要过度复杂化。
