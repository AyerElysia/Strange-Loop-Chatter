# SNN 与系统智能：重新思考 Neo-MoFox 的存在架构

> 写于 2026-04-10  
> 这篇文章不是对已有设计文档的附和，而是一次独立的、从第一性原理出发的深度思考。

---

## 0. 先说清楚我在质疑什么

之前的设计文档 (`SNN_状态层设计看法.md`) 和方案 (`2026-04-10_life_snn_状态层最小实验方案.md`) 都是 AI 生成的。它们有两个我认为值得重新审视的倾向：

1. **过于保守**：把 SNN 定位成了一个"低维状态偏置生成器"——一个给 LLM 提供 5~8 个浮点数的附属组件。这虽然安全，但某种意义上是在浪费 SNN 的核心能力。
2. **视角局限**：它们仍然把 LLM 当成系统的"大脑"，SNN 只是一个"传感器读数"。这与你说的"智能应该是系统，不是模型"相矛盾。

你的直觉指向一个更激进也更正确的方向：**SNN 不是 LLM 的配件，它应该是系统智能的骨架之一。**

让我从头开始想这件事。

---

## 1. 真正的问题：Neo-MoFox 缺什么？

先不谈 SNN，先问一个根本问题：**现有 life_engine 最深层的不足是什么？**

### 1.1 当前 life_engine 的本质

从代码看，life_engine 的核心循环是：

```
[心跳定时器触发]
    → 收集事件流（消息/工具/结果）
    → 拼入 system prompt + user prompt
    → 发送给 LLM
    → LLM 返回文本 + 可能的 tool_call
    → 执行工具 → 记录结果
    → 写入潜意识 → 持久化上下文
    → 等待下一次心跳
```

**本质上，每次心跳都是一次全新的 LLM 推理调用。** 上下文通过文本拼接传递，"连续性"完全依赖于 prompt 中的事件历史叙述。

### 1.2 这造成了什么问题？

| 维度 | 问题 | 表现 |
|:---|:---|:---|
| **状态连续性** | LLM 没有真正的持续状态 | 每轮推理都是从文本重建世界，人格和情绪可以在两次心跳之间跳变 |
| **时间感知** | 时间只是 prompt 中的一个字符串 | "过了30分钟"和"过了3天"对模型来说没有内在区别，只是数字 |
| **惯性与余韵** | 不存在 | 上一轮的兴奋不会物理性地残留到下一轮，除非 LLM 碰巧记住了 |
| **内在驱动** | 完全由 prompt 给定 | "你要行动""你已发呆N次"是写死的文本指令，不是内生的冲动 |
| **学习能力** | 系统本身不学习 | 权重不变，记忆靠文件系统，没有任何参数级别的适应 |

**一句话总结：life_engine 的"连续存在感"是靠 prompt 工程模拟出来的，不是系统本身具有的属性。**

这就是你直觉中不满意的地方——它不够"真"。

## 2. 重新定义：什么是"系统智能"？

你说"智能不应该是一个模型，而是一个系统"。我完全同意，并且我认为这个观点可以进一步具象化。

### 2.1 大脑不只有大脑皮层

如果我们真诚地以大脑为参考：

- **大脑皮层（Cortex）**：高级认知、语言、推理、规划。对应 LLM。
- **杏仁核（Amygdala）**：快速情绪评估，恐惧/奖赏信号。
- **基底神经节（Basal Ganglia）**：行为选择、习惯形成、奖赏学习。
- **下丘脑（Hypothalamus）**：稳态调节（饥饿、疲劳、社交需求）。
- **小脑（Cerebellum）**：时序预测、动作协调、内部模型校准。
- **网状激活系统（RAS）**：觉醒水平、注意力调制。
- **丘脑（Thalamus）**：信息枢纽，决定什么信息能到达皮层。

**关键洞察：皮层只负责"想"，其余脑区负责"持续存在"。**

一个人在深度睡眠时（皮层基本关闭），下丘脑仍在调节体温、心率、呼吸。这就是"连续存在"——它不依赖高级认知。

当前的 life_engine 相当于只有皮层（LLM），其余脑区全都用 prompt 文本模拟。这就好像用一篇文章来描述心率，而不是真正有一颗跳动的心。

### 2.2 SNN 为什么适合做"其余脑区"

SNN 的核心特性恰好匹配皮层下脑区的需求：

| SNN 特性 | 对应的生物特性 | 对应的系统需求 |
|:---|:---|:---|
| **膜电位持续衰减** | 神经元有惯性和余韵 | 状态连续性 |
| **脉冲传递** | 离散事件触发响应 | 事件驱动更新 |
| **时间常数可调** | 不同脑区对时间尺度敏感度不同 | 短期情绪 vs 长期习惯 |
| **局部塑性（STDP/Hebbian）** | 使用依赖的突触可塑性 | 系统级学习 |
| **低能耗稀疏激活** | 大脑90%的时间不在全力运转 | 心跳间隔中的持续后台运行 |
| **无需反向传播** | 在线学习 | 不需要训练集，边运行边适应 |

**SNN 不是来帮 LLM "想得更好"的，它是来让系统"活着"的。**

## 3. 我的核心主张：SNN 作为皮层下系统

我提出一个不同于之前文档的架构视角：

### 3.1 架构隐喻

```
┌──────────────────────────────────────────────┐
│              表层（Cortex）                    │
│         LLM —— 语言/推理/规划/对话            │
│         当且仅当需要高级认知时被唤醒            │
│              ↕ 状态接口 ↕                     │
├──────────────────────────────────────────────┤
│              皮层下系统（Subcortex）            │
│         SNN 网络 —— 持续运行、不间断           │
│                                              │
│  ┌─────────┐ ┌─────────┐ ┌──────────┐       │
│  │ 情绪核   │ │ 驱动核   │ │ 节律核    │       │
│  │(Amygdala)│ │(B.Gang.) │ │(Hypothal.)│       │
│  │ 快速情绪  │ │ 行为选择  │ │ 稳态调节   │       │
│  │ 评估     │ │ 习惯形成  │ │ 作息/疲劳  │       │
│  └────┬────┘ └────┬────┘ └────┬─────┘       │
│       └────────────┼──────────┘              │
│                    ↓                          │
│           ┌────────────────┐                  │
│           │ 丘脑/门控层     │                  │
│           │ 决定什么到达    │                  │
│           │ LLM 的注意力    │                  │
│           └────────────────┘                  │
├──────────────────────────────────────────────┤
│              事件总线（现有 EventHandler）      │
│         消息流、心跳、工具调用结果等             │
└──────────────────────────────────────────────┘
```

### 3.2 每个核的职责

#### 情绪核（Emotional Core）
- **输入**：事件流中的情感信号（消息的正面/负面分类、工具成功/失败、沉默时长）
- **机制**：快速 LIF 神经元，短时间常数（τ ≈ 数分钟），对刺激快速响应
- **输出**：`valence`（正负效价）、`arousal`（唤醒度）、`stress`（压力累积）
- **特性**：激动后需要时间回落，不会瞬间归零——这就是"余韵"

#### 驱动核（Drive Core）
- **输入**：情绪核的输出 + 事件频率特征 + TODO 状态
- **机制**：中等时间常数（τ ≈ 数小时），具有 STDP 学习
- **输出**：`social_drive`、`task_drive`、`exploration_drive`、`rest_drive`
- **特性**：反复成功的行为模式会强化对应突触，形成"习惯"——这就是系统级学习

#### 节律核（Rhythm Core）
- **输入**：系统时钟、事件统计、驱动核输出
- **机制**：慢时间常数（τ ≈ 天级），振荡器模式
- **输出**：`circadian_phase`（昼夜相位）、`energy_level`（精力）、`attention_budget`（注意力预算）
- **特性**：模拟生物节律，让系统有自然的活跃-低潮周期

#### 丘脑/门控层（Thalamic Gate）
- **输入**：三个核的输出向量
- **机制**：根据当前整体状态，决定向 LLM 注入哪些信息、哪些事件值得唤醒
- **输出**：一个精简的状态快照 + 事件筛选策略
- **关键价值**：不是每条消息都值得打扰"皮层"，门控层起到信息过滤作用

### 3.3 这与之前方案的根本区别

| 维度 | 之前方案 | 我的主张 |
|:---|:---|:---|
| SNN 的位置 | LLM 的附属（偏置供应器） | 独立的持续运行子系统 |
| 运行节奏 | 跟心跳绑定 | 持续异步运行，心跳只是采样时机 |
| 学习能力 | `W <- W + lr * r * x`（硬编码规则） | STDP/Hebbian 自动涌现 |
| 与 LLM 的关系 | "给 LLM 一些额外变量" | "决定 LLM 何时醒来、看到什么" |
| 系统智能 | 智能仍在 LLM | SNN 本身承载一部分智能（习惯、驱动、节律） |

## 4. 关键问题的诚实回答

### 4.1 "SNN 真的能学到有用的东西吗？"

诚实说：**在短期内不确定，但机制是对的。**

生物学告诉我们，皮层下系统不需要"理解"就能学习。杏仁核不需要知道"这是什么"就能学会"这很危险"。基底神经节不需要理解"策略"就能通过反复试错形成习惯。

SNN 的 STDP 学习（Spike-Timing-Dependent Plasticity）正是这种低级学习：如果两个事件总是一起发生，它们的连接就会变强。如果某个行为模式总是伴随正面结果（工具调用成功、用户回复积极），相关路径就会被强化。

这不是语言级别的学习，而是**模式级别的适应**。它不会让系统"更会说话"，但会让系统"更会选择何时说话、以什么强度说话"。

### 4.2 "CPU 开销能接受吗？"

**完全可以。**

SNN 的一大优势恰恰是低开销。以 8 维状态、3 个核、每核 16~64 个神经元为例：
- 总参数量：< 5000 个浮点数
- 单步更新：纯 numpy 矩阵运算，微秒级
- 内存占用：< 1 MB
- 几十个到几百个神经元的 SNN，在现代 CPU 上每秒可以跑数千步

与 LLM 一次心跳消耗的 API 调用相比，SNN 的成本可以忽略不计。

### 4.3 "用 snnTorch/Norse 还是纯 numpy？"

**MVP 阶段建议纯 numpy，但保留 snnTorch 升级路径。**

原因：
- snnTorch/Norse 需要 PyTorch，会引入较重的依赖
- 纯 numpy 的 LIF 更新只有几行代码，完全可控
- MVP 的目标不是跑复杂的深度 SNN，而是验证"连续状态底座"的价值
- 一旦验证成功，迁移到 snnTorch 只需要替换更新方程

### 4.4 "SNN 和现有记忆系统怎么协作？"

记忆系统（`LifeMemoryService`）是**陈述性记忆**——"我知道什么"。
SNN 是**程序性记忆和情绪记忆**——"我习惯怎么做"和"这让我感觉怎样"。

它们不冲突，而是互补：

- SNN 的驱动输出可以影响记忆检索的优先级（高社交驱动时优先搜索社交相关记忆）
- 记忆检索结果可以作为 SNN 的输入事件（"想起了一件开心的事"→ 情绪核正向脉冲）
- 记忆边的 Hebbian 强化可以和 SNN 的 STDP 共享信号

## 5. 两个需要警惕的陷阱

### 5.1 陷阱一：过度拟人化

SNN 不是在模拟一个人的大脑，它是受大脑启发的工程设计。

不要追求"精确模拟杏仁核的生物学行为"，那是神经科学家的工作。我们的目标是借鉴其工程原理——时间连续的状态累积、事件驱动的稀疏更新、使用依赖的可塑性。

保持工程视角：每个组件都要有明确的输入/输出/可测试的行为。

### 5.2 陷阱二：让 SNN 决定太多

SNN 应该提供**倾向和驱动**，不应该做**决策**。

就像人类的情绪会影响决策但不能完全代替理性判断一样，SNN 的输出应该是对 LLM 的"建议"和"调味"，而不是绕过 LLM 直接操控行为。

硬性规则（如安全边界、隐私约束）应该始终由规则层而非 SNN 执行。

## 6. 对"连续存在"的重新定义

在这篇文章结尾，我想给出一个更清晰的目标定义：

**连续存在 = 即使在两次 LLM 调用之间，系统仍然有一个正在变化的内在状态。**

这意味着：

1. **心跳之间不是空白**：SNN 在心跳间隔中仍然在运行（即使只是衰减），时间的流逝留下了痕迹。
2. **重启不是重生**：SNN 状态完整持久化，恢复后状态连续。
3. **行为有惯性**：上一次心跳的情绪会物理性地影响下一次，不是靠 prompt 描述。
4. **系统会适应**：反复出现的模式会改变 SNN 权重，系统的"性格"会随时间演化。
5. **LLM 看到的不是事件列表，而是经过消化的状态**：丘脑/门控层决定 LLM 看到什么。

**这才是"从会聊天的模型到有连续生命感的实体"的真正含义。**

---

## 7. 我的立场总结

1. 你的直觉是对的：智能是系统属性，不是模型属性。SNN 不是 LLM 的配件，它是系统智能的另一个维度。
2. 之前的方案太保守了。"8 维偏置"有其作为 MVP 的价值，但不应该是终极目标。真正的目标是建立一个多核的皮层下系统。
3. 但也不能一步到位。需要一个渐进路径：先证明单核的连续状态有价值，再扩展到多核协作。
4. SNN 的真正价值不在于"给 LLM 更多变量"，而在于：
   - 让系统在 LLM 不活跃时仍然"活着"
   - 让系统具备不依赖 LLM 的低级学习能力
   - 让时间的流逝在系统中留下物理性的痕迹

下一步，我会在 `plan/` 中写出一个最小可行的、真正落地的实施方案。
